## 中文
### 概览
`v0.2.9` 覆盖了 `v0.2.8` 之后的全部功能新增与修复，重点集中在 Bot 平台集成、OmniVoice TTS、About 页版本检查、屏幕视觉上下文体验、记忆模型下载稳定性、macOS Live2D 兼容性，以及多语言与构建流程修复。

### 新增功能
- 新增统一 Bot 集成设置；将平台配置、运行状态、消息管线和工具能力集中到新的 Bot 设置页。
- 新增 Bot 平台运行时与后端文本处理管线；支持平台连接管理、消息上下文注入、控制标签解析和角色回复分发。
- 新增 Bot 媒体管线与图像工具；支持处理平台媒体输入，并将图像生成能力接入 Bot 工具流程。
- 新增 OmniVoice TTS Provider；支持在 TTS 设置中配置 OmniVoice，并接入统一语音合成管理流程。
- 新增 OmniVoice 输出语言设置，允许按提供商配置控制合成输出语言。
- 新增 About 页；显示当前版本，并可检查 GitHub Releases 中是否存在新版本。

### 修复
- 修复 OpenAI 工具历史回放，避免工具调用与工具结果在后续上下文中重放不完整或顺序异常。
- 修复自动视觉分析的陈旧上下文写回问题，并避免禁用自动视觉后继续泄漏旧屏幕上下文。
- 修复屏幕 VLM 触发策略；仅在聊天输入聚焦时暂停观察，并允许屏幕上下文持久化时不强制触发回复。
- 修复本地 VLM 请求代理处理，避免本地视觉模型调用被系统代理误导。
- 修复视觉错误展示，让视觉分析失败原因更容易在界面中辨认。
- 修复 macOS 桌宠窗口 Live2D 尺寸和内置 Live2D 模型加载问题。
- 修复记忆模型下载在不支持 Range Header 的环境下失败的问题，并改善失败提示、进度反馈和并行下载恢复能力。
- 修复 Bot token 环境变量默认值、配置迁移日志、控制标签解析，以及相关 clippy 警告。
- 修复繁体中文工具标签和部分本地化文案。
- 修复 macOS 构建中的 Cargo 依赖解析问题。

### 其他更新
- 更新应用图标，并补齐 Android、iOS、Windows 和 macOS 图标资源。
- 优化屏幕上下文历史策略、主动观察评论节奏和相关设置可读性。
- 调整设置页标签对齐与 Bot/Telegram 区域命名。
- 更新 Bot README 描述，并新增仓库 AGENTS 指南。
- 优化 ONNX Runtime 解析，使其在 NixOS 等环境中更稳健。
- 升级应用版本号：`0.2.8 -> 0.2.9`。

### 鸣谢
- 感谢 [@Initsnow](https://github.com/Initsnow) 对本版本工作的支持与贡献。

## English
### Overview
`v0.2.9` includes all additions and fixes since `v0.2.8`, with the main focus on Bot platform integration, OmniVoice TTS, About-page update checks, screen vision context behavior, memory-model download reliability, macOS Live2D compatibility, plus localization and build-flow fixes.

### New Features
- Added unified Bot integration settings, bringing platform configuration, runtime status, message pipelines, and tool capabilities into the new Bot settings page.
- Added Bot platform runtimes and backend text processing, including platform connection management, message context injection, control-tag parsing, and character reply dispatch.
- Added Bot media pipelines and image tooling, enabling platform media input handling and image-generation support in Bot tool flows.
- Added the OmniVoice TTS provider with configuration support in the TTS settings and integration into the shared speech synthesis manager.
- Added an OmniVoice output language setting so synthesized output language can be controlled per provider configuration.
- Added an About page that shows the current version and checks GitHub Releases for newer versions.

### Fixes
- Fixed OpenAI tool history replay so tool calls and tool results are preserved more reliably in follow-up context.
- Fixed stale auto-vision analysis writeback and prevented disabled auto vision from leaking old screen context.
- Fixed screen VLM trigger behavior by pausing observation only while chat input is focused and allowing screen context persistence without forced replies.
- Fixed proxy handling for local VLM requests so local vision model calls are not routed through system proxies.
- Improved vision error display so analysis failures are easier to read in the UI.
- Fixed macOS pet-window Live2D sizing and bundled Live2D model loading.
- Fixed memory-model downloads in environments without Range Header support, with clearer failure messages, better progress feedback, and more resilient parallel transfers.
- Fixed Bot token environment defaults, config migration logging, control-tag parsing, and related clippy warnings.
- Fixed Traditional Chinese tool labels and localization copy.
- Fixed Cargo dependency resolution for macOS builds.

### Other Updates
- Updated the app icon set and added Android, iOS, Windows, and macOS icon assets.
- Refined screen context history policy, proactive observation pacing, and related settings readability.
- Adjusted settings tab label alignment and Bot/Telegram section naming.
- Updated Bot README descriptions and added repository AGENTS guidelines.
- Improved ONNX Runtime resolution for NixOS-style environments.
- Bumped the app version from `0.2.8` to `0.2.9`.

- Thanks to [@Initsnow](https://github.com/Initsnow) for their support and contributions to this release.
