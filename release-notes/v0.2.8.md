## 中文
### 概览
`v0.2.8` 覆盖了 `v0.2.7` 之后的全部功能新增与修复，重点集中在梦境记忆整理、用户资料备份同步、视觉与图像生成提供商增强、繁体中文本地化，以及设置页、桌宠快捷键、聊天错误提示和背景模式的稳定性修复。

### 新增功能
- 新增梦境记忆整理能力；记忆面板增加 Dream 视图，可查看整理摘要、最近任务、待审核提案，并支持手动运行、审核通过或拒绝记忆合并/冲突处理提案。
- 新增梦境记忆配置与迁移；支持每日定时整理、自动应用策略、冲突审核策略，并为旧记忆补齐结构化前缀与新表结构。
- 新增用户资料持久化同步；用户名与用户设定会写入应用数据目录，并在导出、导入和恢复备份时与 `characters.json` 保持一致。
- 视觉模型新增 Anthropic VLM Provider；视觉设置页支持 Anthropic 默认地址、默认模型、API Key、模型列表检测与连通性反馈。
- Stable Diffusion 图像生成新增 Prompt Prefix，可在提供商配置中统一追加正向提示词前缀。
- 新增繁体中文本地化；补充 `README_ZH-TW.md`、繁中界面文案与语言选择入口。

### 修复
- 修复记忆图谱与桌宠设置页的交互体验问题，提升图谱查看、设置项布局和状态反馈稳定性。
- 修复视觉设置里的摄像头预览反馈，避免预览状态与实际摄像头操作不同步。
- 修复聊天错误 toast 处理，统一 Tauri 错误提取逻辑，减少错误信息显示为对象或丢失细节的情况。
- 修复桌宠全局快捷键开关的注册、注销和状态同步问题。
- 修复默认背景模式与背景轮播初始化逻辑，避免未显式选择时落入错误模式。
- 修复上下文消息字符上限文案与后端限制不一致的问题，并补齐相关多语言文案。
- 修复梦境记忆与记忆升级配置相关测试、clippy 警告和边界配置处理。

### 其他更新
- 优化 LLM prompt cache 排序，降低上下文组织中的缓存失效概率。
- 隐藏 async-openai 客户端的无意义警告日志，减少控制台噪音。
- 调整视觉提供商标签，统一 OpenAI-compatible / OpenAI 相关表述。
- 优化设置页布局，覆盖角色管理、API 设置、MCP 设置等区域。
- 更新 README 中的 llama.cpp API 支持说明与记忆系统描述。
- 补充多语言文案，覆盖梦境记忆、Anthropic 视觉、图像生成前缀、桌宠快捷键、上下文窗口和默认背景模式。
- 升级应用版本号：`0.2.7 -> 0.2.8`。

## English
### Overview
`v0.2.8` includes all additions and fixes since `v0.2.7`, with the main focus on dream memory consolidation, user-profile backup synchronization, vision and image-generation provider improvements, Traditional Chinese localization, plus stability fixes for settings, pet shortcuts, chat error toasts, and background mode handling.

### New Features
- Added dream memory consolidation; the memory panel now includes a Dream view for summaries, recent jobs, pending proposals, manual runs, and approving or rejecting memory merge/conflict proposals.
- Added dream memory configuration and migrations, including daily scheduled consolidation, auto-apply policy, conflict-review policy, structured-prefix backfill, and new dream-memory tables.
- Added persistent user-profile synchronization; username and user persona are saved to the app data directory and kept consistent with `characters.json` during export, import, and restore.
- Added Anthropic VLM provider support for vision, including default endpoint/model values, API key handling, model-list detection, and connectivity feedback in the vision settings page.
- Added Stable Diffusion prompt prefix support so provider-level positive prompt prefixes can be applied consistently during image generation.
- Added Traditional Chinese localization, including `README_ZH-TW.md`, UI translations, and language-selection support.

### Fixes
- Fixed memory graph and pet settings UX issues, improving graph readability, settings layout, and state feedback.
- Fixed camera preview feedback in vision settings so preview state better matches actual camera operations.
- Fixed chat error toast handling by normalizing Tauri error extraction and preventing object-shaped or incomplete error messages.
- Fixed pet global shortcut registration, unregistering, and state synchronization.
- Fixed default background mode and slideshow initialization when no explicit mode has been selected.
- Fixed context message character-limit copy so the frontend text matches backend behavior, with localized copy updates.
- Fixed dream-memory and memory-upgrade configuration tests, clippy warnings, and edge-case config handling.

### Other Updates
- Optimized LLM prompt-cache ordering to reduce cache misses during context assembly.
- Silenced non-actionable async-openai client warnings to reduce console noise.
- Refined vision provider labels for OpenAI-compatible / OpenAI wording.
- Polished settings layout across character management, API settings, and MCP settings.
- Updated README coverage for llama.cpp API support and the memory-system description.
- Added localized copy for dream memory, Anthropic vision, image prompt prefixes, pet shortcuts, context windows, and default background mode.
- Bumped the app version from `0.2.7` to `0.2.8`.
